🚀 New project: **AI-Powered Customer Feedback Automation Workflow**

Coming from a customer-insights / VOC analytics background, I wanted to show what
it looks like to turn a familiar business problem into a real **AI automation
pipeline** — not just a notebook.

The problem: support teams get hundreds of reviews across app, web, and email.
Reading and tagging them by hand is slow and inconsistent, so emerging issues
get caught late.

What I built:
✅ An **n8n** workflow that runs manually or on a daily schedule
✅ A **Python (pandas)** pipeline that cleans the data and classifies every
review by **sentiment, issue category, and urgency**
✅ A summary report with **sentiment distribution, top complaints, emerging
issues, and recommended actions**
✅ A **pluggable classifier** — rule-based by default (free, offline) with an
**optional OpenAI** upgrade and automatic fallback
✅ Logging, error handling, and **dashboard-ready CSV** output (Power BI / Sheets)

The part I enjoyed most: designing it like a real automation — structured
outputs, exit codes the workflow can branch on, and documentation so someone
else could actually run it.

This is me bridging analytics → AI automation engineering, one practical project
at a time. Code + full README on GitHub 👇
